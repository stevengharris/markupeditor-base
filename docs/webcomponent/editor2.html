The initial HTML contents of this editor are fetched from filename, editor2.html.


<markup-editor 
    filename="editor2.html" 
    userscript="editor2.js" 
    userstyle="editor2.css" 
    delegate="MyDelegate" 
    toolbar="MyToolbarConfig" 
    behavior="MyBehaviorConfig"
    muscript="../src/markupeditor.umd.js" 
    mustyle="../styles/markupeditor.css">
</markup-editor>

The userscript file, editor2.js, defines the classes for MyToolbarConfig, MyBehaviorConfig, and MyDelegate, and registers instances of them so they can be looked up the names passed in delegate, toolbar, and behavior.

  • MyToolbarConfig removes the image and table items from the "insert bar", leaving only the ability to insert links. It adds underline in as a formatting option. It drops the "search" item and adds in undo/redo (the "correction bar"), which is handy on mobile devices.

  • MyBehaviorConfig stops this editor from taking focus when the page is loaded, leaving the editor above with focus.

  • MyDelegate in this case doesn't do much, other than log to the console as you type into this editor. The MarkupDelegate will receive callbacks as the document you're editing changes and is the simplest way to "hook into" the editor to react to those changes.
