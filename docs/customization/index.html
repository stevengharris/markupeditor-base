MarkupEditor Customization Demo

The simplest way to embed WYSIWYG editing in your web site is to use the MarkupEditor web component. The web component makes sure the MarkupEditor script is only loaded once, and that the base MarkupEditor styling is applied (e.g., that handles the way the toolbar looks, font sizing, etc) to the editor within the web component without potential side-effects to the page you embed it in. Using the <markup-editor> web component attributes, you have access to all of the customization features of the base MarkupEditor, including:

  • Provide initial HTML contents for editing directly or by a file name.

  • Specify placeholder text for when the document is empty.

  • Provide your own scripting and styling within the editor.

  • Provide a custom MarkupDelegate to receive document change notifications.

  • Customize toolbar visibility, contents, element ordering, and augment toolbar contents.

  • Customize key mapping or use hot key editing defaults.

  • Customize behavior, such as whether local images can be embedded, and whether to take focus immediately.

  • Identify the location for the MarkupEditor base script and styling if needed.

If the MarkupEditor base script and styling (markupeditor.umd.js and markupeditor.css) and the web component definition (markup-editor.js) reside in the same directory as your document, and you want a full page WYSIWYG editor with a default toolbar, you can embed the MarkupEditor web component in just two lines: 


    <markup-editor></markup-editor>
    <script src="markup-editor.js"></script>

Default Web Component

This <markup-editor> element uses the default toolbar and specifies editor1.css in the userstyle attribute. In this case, the editor1.css styling is used to set the border and height of the <markup-editor> element. 


<markup-editor 
    userstyle="editor1.css" 
    muscript="../src/markupeditor.umd.js" 
    mustyle="../styles/markupeditor.css">
</markup-editor>

The demos share a common MarkupEditor base script (markupeditor.umd.js) and styling (markupeditor.css), so we also have to specify their location in muscript and mustyle.
