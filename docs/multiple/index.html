Multiple Markup Editors On A Page

Use the markup-editor web component, specifying any script or styling that is specific to that element. The web component makes sure the MarkupEditor script is only loaded once, and that the base MarkupEditor styling is applied (e.g., that handles the way the toolbar looks, font sizing, etc) to the editor within the web component.

This <markup-editor> element uses the default toolbar and specifies editor1.css in the userstyle attribute. In this case, the editor1.css styling is used to set the border and height of the <markup-editor> element. 


<markup-editor userstyle="editor1.css" muscript="../src/markupeditor.umd.js" mustyle="../styles/markupeditor.css">

The demos share common MarkupEditor base script (markupeditor.umd.js) and styling (markupeditor.css), so we also have to specify their location in muscript and mustyle.

This <markup-editor> element uses a userscript (setup.js) to define a MarkupDelegate class (MyDelegate) and ToolbarConfig class (MyToolbarConfig). The delegate and toolbar config are identified by name in the <markup-editor> element attributes. 


<markup-editor userscript="setup.js" userstyle="editor2.css" delegate="MyDelegate" toolbar="MyToolbarConfig" muscript="../src/markupeditor.umd.js" mustyle="../styles/markupeditor.css">

MyToolbarConfig in setup.js removes the image and table items from the "insert bar", leaving only the ability to insert links. It adds underline in as a formatting option. It drops the "search" item and adds in undo/redo (the "correction bar"), which is handy on mobile devices.

MyDelegate in this case doesn't do much, other than log to the console as you type into this editor. The MarkupDelegate will receive callbacks as the document you're editing changes and is the simplest way to "hook into" the editor to react to those changes.
